Compiling from source on Windows

Important

Regular users are recommended to download an install a release version, rather than compiling from source. Releases are published at https://github.com/axoloti/axoloti/releases.

MSYS2

Download and install msys2-i686 from http://msys2.github.io/. Use the default settings. Follow the instructions on the webpage.

Install JDK

Download and run the JDK installer from http://www.oracle.com/technetwork/java/javase/downloads/index.html pick "jdk-8u...-windows-i586.exe" For developers, I recommend downloading and installing the Netbeans with JDK package from http://www.oracle.com/technetwork/java/javase/downloads/jdk-netbeans-jsp-142931.html

Get Axoloti

Clone the axoloti repository using the github windows client, or using the command line (the MSYS2 shell provides git command line tools).

C:\Users\joe> git clone https://github.com/axoloti/axoloti.git
C:\Users\joe> cd axoloti
C:\Users\joe\axoloti> git submodule update --init --recursive

Compile Axoloti

Run the "build.bat" script in the platform_win directory. This will download, extract, and compile all necessary tools.

C:\Users\joe\axoloti> cd platform_win
C:\Users\joe\axoloti\platform_win> build.bat
C:\Users\joe\axoloti\platform_win> cd ..
 Note: the dfu-util executable build with msys2 does not work properly... The windows runtime contains a working libusb + dfu-util.

Launching Axoloti

C:\Users\joe\axoloti> axoloti.bat
