Using the STM32f4Discovery board

Axoloti is designed to run with the Axoloti Core board. But the STM32f4Discovery can be made to work too. Connect both the mini USB port and the micro USB port to your computer.

Modify the firmware

  • in firmware/axoloti_defines.h, uncomment the line "#define BOARD_STM32F4DISCOVERY 1", and comment the line "#define BOARD_AXOLOTI_V03 1"
  • recompile the firmware
  • flash the firmware (using DFU mode)

DFU Mode

For firmware flashing, DFU-mode for flashing can be enabled - not by pressing S1 as documented for the Axoloti Core - but by connecting the BOOT0 pin to VDD, and the PB2 pin to GND during powerup or reset. You can use the jumpers from the backside of the kit for this purpose. After flashing, remove the jumpers again from BOOT0 and PB2, and press the reset button.

Limitations

  • SDcard cannot be used (pin conflict with the audio DAC)
  • GPIO pins PA4, PA5, PA6, PA7 and PC0 should not be used - they're not free for user I/O on the Discovery board.
  • no audio input available
  • the on-board microphone is not supported
  • the on-board accelerometer is not supported
  • the on-board LEDs are not supported
  • 48kHz sample rate is a bit off : 47991.07031kHz (0.32 cent lower tuning)
